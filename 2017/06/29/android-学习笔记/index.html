Service

bindService

Exception:
android.app.ServiceConnectionLeaked: Activity com.remote.client.RemoteClientActivity has leaked ServiceConnection   
com.remote.client.RemoteClientActivity$1@44eb1200   
that was originally bound here

Why:

产生该异常的原因是在activity销毁的时候，没有调用unbindService。

Fix:

protected void onDestroy() {  
    unbindService(serviceConnection);  
    super.onDestroy();  
}

Service 和 Activity 通信

Activity通过Intent启动Service，service在后台运行，当Service执行完某些操作之后我们希望UI线程，这就是Service和Activity之间的通信问题。

作者@xiaanming的一篇文章就很好地介绍这个问题：Android Service与Activity之间通信的几种方式。

文章中介绍的关于Binder对象完成Service和Activity通信的代码部分有些问题，即在MsgService类的startDownload()方法中，开启新Thread，并在新Thread中执行回调函数，而回调函数onProgress()要更新的UI组件是在Activity的UIThread里创建的，直接更新会出现异常。因此要修改Activity在注册回调接口时的代码，保证执行UI更新部分的代码时切换到UIThread，修改如下：

// 注册 [localService实例] 中的 [回调接口]
localService.setOnProgressListener(new OnProgressListener() {
    @Override
    public void onProgress(int progress) {
        runOnUiThread(new Runnable() {
            @Override
            public void run() {
                secondDisplay.setText(progress + "");
            }
        });
    }
});
